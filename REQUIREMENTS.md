# REQUIREMENTS.md

## What We Are Building
A web app where the user pastes a Wikipedia URL, gets a summary, and can chat with the article using a local LLM (RAG).

## Functional Requirements
- User pastes a Wikipedia URL and submits
- App fetches and parses the article
- App shows a summary generated by local LLM
- User can ask questions answered from the article content only (RAG)
- Error shown for invalid URLs or fetch failures

## Non-Functional Requirements
- LLM runs locally via Ollama (no OpenAI/Anthropic)
- Embeddings via Ollama nomic-embed-text (local)
- Vector DB: Qdrant running in Docker
- Full stack starts with: docker compose up
- 85%+ test coverage

## Out of Scope
- Auth, user accounts, multi-article history, analytics, UI polish

## Assumptions
- Wikipedia REST API used (structured JSON, better than raw HTML scraping)
- Model: llama3.2:3b (runs on low-end laptops)
- Chunk size: 500 tokens max, section-aware (Wikipedia sections are natural topic boundaries)
- Top 5 chunks retrieved per query
- Each new article replaces the previous one in Qdrant
